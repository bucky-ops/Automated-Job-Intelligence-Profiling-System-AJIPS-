## Decision Making Process

1. **Minor Changes**: Any maintainer can approve and merge
2. **Major Changes**: Require approval from Project Lead
3. **Breaking Changes**: Require discussion and consensus
4. **Security Issues**: Immediate action by Project Lead or Security Lead

## Becoming a Maintainer

We welcome new maintainers! To become a maintainer:

1. **Contribute consistently** for at least 3 months
2. **Demonstrate expertise** in a specific area
3. **Show good judgment** in code reviews and discussions
4. **Be active** in the community
5. **Be nominated** by an existing maintainer

Current maintainers will vote on new maintainer applications.

## Code Review Guidelines

### For Reviewers

- Be constructive and kind
- Explain the "why" behind suggestions
- Approve when changes meet standards
- Request changes for significant issues
- Merge only when CI passes

### Review Checklist

- [ ] Code follows style guidelines
- [ ] Tests are included and passing
- [ ] Documentation is updated
- [ ] Security considerations addressed
- [ ] Performance impact assessed
- [ ] Backward compatibility maintained (or documented)
